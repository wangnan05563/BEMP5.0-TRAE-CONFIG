# BEMP 项目 SonarQube 问题分类与判断标准

## 问题分类体系

### 一、按质量维度分类

#### SECURITY（安全类）

| 规则ID | 规则名称 | BEMP场景说明 |
|--------|----------|-------------|
| secrets:S6702 | 硬编码密钥/Token | 检查 bemp-deploy.yml 等配置文件中的敏感信息 |
| java:S2068 | 硬编码密码 | 检查数据库连接、API认证等 |
| java:S2078 | SQL注入 | 检查 MyBatis XML 中的 ${} 拼接 |
| java:S5146 | Open Redirect | 检查 Controller 中的重定向 |
| java:S5042 | 路径遍历 | 检查文件操作相关代码 |

#### RELIABILITY（可靠性类）

| 规则ID | 规则名称 | BEMP场景说明 |
|--------|----------|-------------|
| java:S2259 | 空指针异常 | 检查 DTO 取值、DAO 查询结果未判空 |
| java:S2095 | 资源泄露 | 检查 IO 流、数据库连接未关闭 |
| java:S2274 | 并发修改 | 检查 Service 层的线程安全问题 |
| java:S2696 | 实例方法修改静态字段 | 检查 Service 中对静态变量的修改 |

#### MAINTAINABILITY（可维护性类）

| 规则ID | 规则名称 | BEMP场景说明 |
|--------|----------|-------------|
| java:S3776 | 认知复杂度过高 | Service 方法拆分，提取私有方法 |
| java:S1192 | 字符串重复定义 | 提取为常量或配置项 |
| java:S1118 | 工具类添加私有构造 | 已在 Util 类中遵循 |
| java:S100 | 方法命名规范 | 检查下划线命名 |
| java:S1135 | TODO标记 | 记录待办事项 |

### 二、按BEMP项目上下文分类

#### 真实问题（必须修复）

特征：
- 直接影响系统安全性或可靠性
- 违反 Java 编码基本规范
- 不依赖 BEMP 框架特性即可判断

示例：
- 空指针未检查
- SQL注入风险
- 硬编码敏感信息
- 资源泄露

#### 框架误报（可标记 falsepositive）

特征：
- 由 BEMP 框架注解/模式触发
- 代码在 BEMP 运行时环境下是安全的
- SonarQube 不理解框架的依赖注入机制

常见误报模式：

| 规则 | 触发场景 | 误报原因 |
|------|----------|----------|
| java:S112 | 抛出 Exception 而非子类 | BEMP 框架统一使用 BempRuntimeException |
| java:S116 | 字段命名不符合规范 | DTO 字段名需与数据库列名对应 |
| java:S1144 | 未使用的私有方法 | 可能被 @CloudReference 代理调用 |
| java:S1149 | 使用 Hashtable | 框架内部兼容性需要 |

标记 falsepositive 的 comment 模板：
```
BEMP框架误报：[说明原因]。该代码在BEMP运行时环境下[解释为何安全/正确]。
```

#### 可接受债务（标记 accept）

特征：
- 业务需要但不符合最佳实践
- 修复成本高于收益
- 已有补偿措施

标记 accept 的 comment 模板：
```
可接受技术债务：[说明业务原因]。补偿措施：[已有的替代方案]。
```

### 三、问题严重级别判定矩阵

| 严重级别 | 安全类 | 可靠性类 | 可维护性类 |
|----------|--------|----------|-----------|
| BLOCKER | 密码泄露、SQL注入 | 数据丢失风险 | - |
| HIGH | 敏感信息暴露 | 空指针崩溃、资源泄露 | 认知复杂度严重超标 |
| MEDIUM | 潜在XSS | 异常处理不当 | 字符串重复、命名不规范 |
| LOW | 信息泄露 | 边界条件 | TODO标记、注释缺失 |
| INFO | - | - | 代码风格建议 |

### 四、BEMP 特有检查项

以下检查项不在 SonarQube 标准规则中，但对 BEMP 项目质量至关重要：

1. **事务注解检查**：所有修改操作必须有 `@Transactional(rollbackFor = Exception.class)`
2. **AOP 切面覆盖**：Controller 方法应被切面统一处理 legalNo
3. **个性化路径检查**：API 路径必须包含 `/banks/hnnxbank/` 前缀
4. **常量提取**：状态码、类型码等必须定义为静态常量
5. **日志规范**：关键业务操作必须有 logger.info 日志
6. **DTO 字段类型**：金额类字段 Entity 用 BigDecimal，DTO 用 String
