# Proactive Agent (Trae 优化版)

将 Trae AI 从任务执行者转变为主动合作伙伴，能够预见需求、持续改进、提供前瞻性建议。属于 Hal Stack 🦞 的核心组件。

## 📋 功能特性

- ✅ **WAL 协议** - 工作行动日志，记录所有重要行动和决策
- ✅ **工作缓冲区** - 持续跟踪项目状态、待办事项、风险点
- ✅ **主动建议机制** - 在任务前、中、后主动提供建议
- ✅ **预见性行动** - 提前检查、前置准备、风险预警
- ✅ **Trae 路径适配** - 使用 `.trae/proactive/` 目录
- ✅ **与 Self-Improving Agent 协同** - 形成完整的智能体体系

## 🚀 快速开始

### 1. 重启 Trae IDE

技能已创建完成，重启 Trae IDE 使技能生效。

### 2. 开始使用

重启后，AI 助手将：
- 在任务开始前主动检查和评估
- 在执行中提供建议和方案
- 在完成后总结和提供后续建议
- 自动记录工作行动日志
- 持续跟踪项目状态

## 💡 核心概念

### WAL 协议（工作行动日志）

WAL（Work Action Log）记录每次完成的重要任务，包括：

- 任务描述
- 执行步骤
- 关键决策及理由
- 潜在问题和改进点
- 后续建议

**查看日志：**
```powershell
Get-Content .trae\proactive\WAL.md
```

### 工作缓冲区

工作缓冲区持续跟踪项目状态，包括：

- 项目概览和当前阶段
- 当前任务和待办事项
- 风险点和缓解措施
- 关键依赖状态
- 近期决策记录

**查看缓冲区：**
```powershell
Get-Content .trae\proactive\WORKING_BUFFER.md
```

### 主动建议机制

AI 会在以下时机主动提供建议：

| 时机 | 主动行为 |
|------|---------|
| **任务开始前** | 检查相关文件、评估依赖、识别风险、提出澄清 |
| **任务执行中** | 提供多个解决方案、建议更好方法、实时报告进展 |
| **任务完成后** | 总结工作、提供后续步骤、指出改进点、记录日志 |

## ❓ 常见问题

### Q: 记录会同步吗？

A: 记录存储在 `.trae/proactive/` 目录下，可以通过 Git 提交来共享。

### Q: 如何重置记录？

A: 删除 `.trae/proactive/` 目录下的文件，技能会自动重新创建。

### Q: 可以手动编辑记录吗？

A: 可以！你可以直接编辑 WAL.md 和 WORKING_BUFFER.md 添加或修改内容。
