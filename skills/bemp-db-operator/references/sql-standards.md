# SQL编写与执行标准

## 1. SQL编写规范

### 1.1 通用规范

| 规则 | 说明 | 示例 |
|------|------|------|
| 关键字大写 | SELECT, INSERT, UPDATE, DELETE | `SELECT * FROM` |
| 表名字大写 | 表名使用大写 | `TM_AUTHORITY` |
| 字段名大写 | 字段名使用大写 | `LEGAL_NO` |
| 缩进2空格 | 代码块缩进 | |
| 语句末分号 | 每条语句以分号结束 | `;` |

### 1.2 增量SQL规范

所有增量SQL必须遵循**"先删除后新增"**策略：

```sql
-- 先删除
DELETE FROM TM_AUTHORITY WHERE ID IN (${ID_LIST});

-- 后新增
INSERT INTO TM_AUTHORITY (...) VALUES (...);

COMMIT;
```

### 1.3 DDL规范

- 建表前必须判断表是否存在
- 新增字段前必须判断字段是否存在
- 创建索引前必须判断索引是否存在
- 创建序列前必须判断序列是否存在

### 1.4 DML规范

- DELETE/UPDATE 必须有 WHERE 条件
- INSERT 必须明确列出字段名
- 脚本末尾必须有 COMMIT
- 批量操作需预估影响行数

## 2. SQL执行标准

### 2.1 执行前检查清单

| 序号 | 检查项 | 要求 |
|------|--------|------|
| 1 | 语法正确 | 无语法错误 |
| 2 | WHERE条件 | DELETE/UPDATE必须有 |
| 3 | 幂等性 | 可重复执行不报错 |
| 4 | 事务完整性 | 末尾有COMMIT |
| 5 | 表/字段存在 | 目标对象存在 |
| 6 | 影响范围 | 行数在预期内 |
| 7 | ID/KEY冲突 | 无冲突 |

### 2.2 执行顺序

```
1. DDL脚本（表结构）
2. DDL脚本（索引）
3. DML脚本（菜单）
4. DML脚本（业务参数）
5. DML脚本（字典）
6. DML脚本（流程编排）
7. DML脚本（待办任务）
```

### 2.3 执行超时设置

| 操作类型 | 超时时间 | 说明 |
|----------|----------|------|
| DDL | 60秒/语句 | 建表、加字段等 |
| DML | 30秒/语句 | 增删改数据 |
| 查询 | 120秒 | SELECT查询 |
| 连接 | 30秒 | 建立连接 |

## 3. SQL复核标准

### 3.1 自动复核项

| 复核项 | 复核方法 | 通过标准 |
|--------|----------|----------|
| 表结构验证 | describe_table | 结构符合预期 |
| 数据验证 | execute_query | 数据内容正确 |
| 行数验证 | SELECT COUNT(*) | 行数变化符合预期 |
| 关联验证 | 关联表查询 | 关联数据无异常 |

### 3.2 人工复核项

| 复核项 | 复核方法 | 责任人 |
|--------|----------|--------|
| SQL逻辑正确性 | 代码审查 | 开发人员 |
| 业务数据合理性 | 业务验证 | 业务人员 |
| 影响范围评估 | 影响分析 | DBA |

## 4. 回退标准

### 4.1 回退触发条件

| 条件 | 回退级别 | 说明 |
|------|----------|------|
| 单条SQL执行失败 | statement | 仅回退失败语句 |
| 脚本内多条SQL关联失败 | script | 回退整个脚本 |
| 复核结果不通过 | script/full | 根据影响范围决定 |
| 业务需求变更 | full | 恢复到执行前状态 |

### 4.2 回退SQL生成规则

| 原操作 | 回退操作 | 数据来源 |
|--------|----------|----------|
| INSERT | DELETE | 使用INSERT的ID列表 |
| DELETE | INSERT | 使用执行前快照 |
| UPDATE | UPDATE | 使用执行前快照 |
| CREATE TABLE | DROP TABLE | 无需额外数据 |
| ALTER TABLE ADD | ALTER TABLE DROP | 无需额外数据 |
| CREATE INDEX | DROP INDEX | 无需额外数据 |

### 4.3 回退验证

回退后必须验证：
1. 数据已恢复到执行前状态
2. 表结构与执行前一致
3. 关联数据无异常
