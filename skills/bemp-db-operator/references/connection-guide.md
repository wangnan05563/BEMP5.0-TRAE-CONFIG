# 数据库连接规范

## 1. 连接方式

### 1.1 Oracle MCP 连接（推荐）

通过 Oracle MCP 工具进行数据库操作，提供标准化的连接管理：

| 工具 | 用途 | 权限 |
|------|------|------|
| `mcp_oracle-mcp_list_schemas` | 列出Schema | 只读 |
| `mcp_oracle-mcp_list_tables` | 列出表 | 只读 |
| `mcp_oracle-mcp_describe_table` | 查看表结构 | 只读 |
| `mcp_oracle-mcp_execute_query` | 执行查询 | 只读 |

**连接参数**：

| 参数 | 开发环境 | 测试环境 | 生产环境 |
|------|----------|----------|----------|
| 主机 | 10.20.18.177 | 待配置 | 待配置 |
| 端口 | 1521 | 1521 | 1521 |
| 服务名 | orcl | 待配置 | 待配置 |
| 用户名 | bemp_hnnx | 待配置 | 待配置 |
| Schema | BEMP_HNNX | 待配置 | 待配置 |

### 1.2 SQL*Plus 连接

```bash
sqlplus bemp_hnnx/123456@10.20.18.177:1521/orcl

```
jdbc:oracle:thin:@10.20.18.177:1521:orcl

## 2. 连接管理规范

### 2.1 连接生命周期

```
建立连接 → 连接测试 → 执行操作 → 结果验证 → 释放连接
```

### 2.2 连接池配置

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 初始连接数 | 1 | MCP单次操作 |
| 最大连接数 | 3 | 并发操作上限 |
| 连接超时 | 30秒 | 建立连接超时 |
| 空闲超时 | 300秒 | 连接空闲回收 |
| 最大生命周期 | 1800秒 | 连接最大存活时间 |

### 2.3 连接失败处理

| 失败类型 | 处理方式 | 重试策略 |
|----------|----------|----------|
| 网络超时 | 检查网络连通性 | 最多3次，间隔5秒 |
| 认证失败 | 检查用户名密码 | 不重试，修正配置 |
| 服务不可用 | 检查Oracle服务状态 | 最多3次，间隔10秒 |
| Schema不存在 | 检查Schema名称 | 不重试，修正配置 |

## 3. 安全规范

### 3.1 密码管理

- 开发环境密码可明文存储于 `config/db-config.json`
- 测试/生产环境密码必须加密存储
- 密码不得出现在日志、报告等输出文件中
- 定期更换数据库密码

### 3.2 权限控制

- Oracle MCP 连接使用只读权限，仅支持 SELECT 语句
- DDL/DML 操作需通过审批后执行
- 生产环境操作需双人复核

### 3.3 操作审计

- 所有数据库操作必须记录日志
- 操作日志包含：时间、操作人、SQL语句、影响行数
- 日志保留期限不少于6个月

---

## 4. 环境切换

### 4.1 切换流程

1. 修改 `config/db-config.json` 中的 `defaultEnvironment` 值
2. 确认目标环境的连接参数已正确配置
3. 执行连接测试验证新环境可用
4. 记录环境切换操作日志

### 4.2 注意事项

- 切换到生产环境前必须获得审批
- 生产环境默认启用 dryRun 模式
- 生产环境回退策略必须为 script 级别以上
