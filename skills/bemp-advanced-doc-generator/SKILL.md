### 1.4 技能职责

- 解析参考 Word 文档的格式规范、内容结构、图表样式
- 扫描 BEMP 工作空间，识别目标业务功能相关代码
- 生成包含 8 章节的标准详细设计 Word 文档
- 生成包含 8 章节的测试用例文档
- 生成包含 6 章节的测试报告文档
- 支持同时输出 Markdown 和 Word 两种格式
- 确保文档格式与参考模板完全一致
- 根据业务场景自动识别并生成流程图或思维导图
- 集成 ProcessOn MCP API 实现可视化文档生成
- 提供自定义模板系统支持用户个性化需求
- 实现错误处理和重试机制保障生成可靠性
- 提供交互式界面和命令行两种使用方式
- 智能识别外围接口与产品接口的字段映射关系
- 专门适配消息转换器/Adapter接口开发场景

## 4. 输出标准

### 4.1 文档格式要求

| 项目 | 标准 |
|------|------|
| 文件格式 | Markdown (.md) 和 Word (.docx) |
| 页面设置 | A4 纵向，边距上下 2.54cm，左右 3.17cm |
| 标题样式 | 一级黑体三号，二级黑体四号，三级黑体小四号 |
| 正文字体 | 宋体小四号，1.5 倍行距 |
| 表格样式 | 表头黑体五号加粗，内容宋体五号 |
| 图表编号 | 图 X-Y、表 X-Y |

### 4.2 文档内容要求

#### 详细设计文档

| 章节 | 必需元素 |
|------|----------|
| 系统概述 | 业务背景≥3 行、设计目标≥2 条、范围说明≥2 条 |
| 功能模块 | 模块划分≥3 个、职责描述完整 |
| 业务流程 | 流程图≥1 张、时序图≥2 张、关键节点说明≥5 条 |
| 数据模型 | ER 图≥1 张、数据字典≥10 条、字段映射关系表（V3.0 新增） |
| 接口定义 | API 接口≥3 个、参数说明完整、消息转换器配置说明（V3.0 新增） |
| 异常处理 | 错误码≥5 个、处理流程图≥1 张 |
| 安全策略 | 安全措施≥3 条 |
| 技术实现 | 核心代码≥5 行、注释完整、开发规范说明（V3.0 新增） |

#### 测试用例文档

| 章节 | 必需元素 |
|------|----------|
| 引言 | 编写目的≥2 条、背景说明≥3 行 |

| 测试环境 | 硬件、软件、测试工具说明完整 |
| 功能测试 | 正常场景≥5 条、异常场景≥3 条、边界值≥3 条、字段映射验证≥5 条（V3.0 新增） |
| 集成测试 | 端到端测试≥2 条、服务调用测试≥2 条（V3.0 新增） |
| 性能测试 | 负载测试、压力测试、稳定性测试各≥1 条 |
| 安全测试 | 认证、授权、加密测试各≥1 条 |

#### 测试报告文档（V3.0 新增）

| 章节 | 必需元素 |

### 7.7 故障排除

| 问题 | 解决方案 |
|------|----------|
| docx 模块未找到 | 执行 `npm install docx` |
| 模板文件不存在 | 检查 assets 目录下是否有对应的 JSON 文件 |
| 参考文档不存在 | 检查 assets 目录下是否有参考 Word 文档 |
| 文档生成失败 | 检查 Node.js 版本是否≥12 |
| 中文显示异常 | 确保使用 SimSun/SimHei 字体 |
| 文件写入失败 | 检查输出路径是否有写入权限 |
| ProcessOn API 连接失败 | 检查 PROCESSON_API_KEY 环境变量是否设置 |
| MCP 调用超时 | 增加重试次数或检查网络连接 |

### 7.8 错误码说明

| 错误码 | 说明 | 解决方案 |
|--------|------|----------|
| -32001 | MCP 调用超时 | 增加重试次数或检查网络连接 |
| EACCES | 文件权限错误 | 检查输出目录权限 |
| ENOENT | 文件不存在 | 检查文件路径是否正确 |
| ECONNREFUSED | 连接被拒绝 | 检查 ProcessOn 服务是否启动 |

### 9.5 场景识别算法

**流程图特征关键词**：
- 流程、步骤、顺序、判断、决策、分支、循环
- 开始、结束、处理、输入、输出、条件、执行
- mermaid、flow、sequence、时序、泳道、bpmn

**思维导图特征关键词**：
- 思维导图、中心主题、分支、层级、概念、知识
- 结构、组织、大纲、树状、树形、金字塔、分类

**消息转换器/Adapter场景特征关键词**（V3.0 新增）：
- MessageConverter、消息转换器、报文转换
- XML转JSON、JSON转XML、fromMessage、toMessage
- 外围接口、产品接口、字段映射、ECIF
- getFunctionIdMapping、广播消息、PICE
- ecif、credit、channel、adapter

**识别逻辑**：
1. 统计关键词匹配得分
2. 根据内容长度和结构加权
3. 检测明显的格式标记
4. 综合评分决定图表类型和文档类型

### 9.6 消息转换器/Adapter接口开发场景适配（V3.0 新增）

**适用场景**：
- 开发新的 MessageConverter 类
- 外围系统接口对接
- 字段映射关系定义
- 报文格式转换开发

**文档生成特点**：
- 详细设计文档增加"字段映射关系"专节
- 详细设计文档增加"消息转换器配置"专节
- 详细设计文档增加"开发规范"专节
- 测试用例增加"字段映射验证"测试集
- 测试用例增加"集成测试"章节
- 测试报告增加"字段映射验证结果"

**字段映射识别规则**：
1. 扫描外围接口规范文档（如 0402006广播机构客户合并.md）
2. 扫描产品服务接口DTO类（如 Ecif4001ReqDto.java）
3. 自动比对字段名称、类型、长度
4. 生成字段映射关系表
5. 标注缺失字段和默认值设置
