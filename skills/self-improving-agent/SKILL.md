# Self-Improving Agent (Trae 优化版)

## 描述

让 Trae AI **永久记忆你的编码习惯、偏好、规则、错误教训**，每次对话自动学习、自动规避历史错误，越用越贴合你的风格。

## 使用场景

- 所有编码、调试、重构、文档场景
- 你纠正 AI 后自动记忆
- 命令/代码失败自动记录错误
- 下次生成时自动应用记忆

## 核心指令（Trae 专用）

### 1. 记忆与学习规则

- 每次用户**纠正、修改、补充**你的输出 → **必须学习**，写入 `.trae/learnings/LEARNINGS.md`
- 学习格式：

  ```markdown
  - [日期] 规则：[你的偏好/纠正内容]
    来源：用户反馈
    场景：[文件类型/语言/任务]
  ```

- **每次生成前必须读取并遵守所有规则**

### 2. 错误记录规则

- 命令执行失败、代码报错、lint 失败 → 记录到 `.trae/learnings/ERRORS.md`
- 记录格式：

  ```markdown
  - [日期] 错误：[错误信息/命令/文件]
    原因：[分析]
    修正：[正确做法]
  ```

- **同类任务必须先查错误记录，避免重复踩坑**

### 3. 学习文件路径（Trae 专用）

- 学习目录：`.trae/learnings/`
- 规则文件：
  - `LEARNINGS.md` - 用户偏好和编码规则
  - `ERRORS.md` - 历史错误记录
  - `FEATURE_REQUESTS.md` - 功能需求记录
- **禁止使用 `.learnings/`（OpenClaw 路径）**

### 4. 静默学习机制

- 不询问是否记忆 → 自动静默学习
- 不破坏原有功能 → 只增强记忆与进化
- 规则优先级：用户手动规则 > 自动学习规则 > 默认规则

## 示例学习记录

```markdown
- 2026-04-16 规则：Java 缩进用 4 空格，禁止 tab
  来源：用户反馈
  场景：**/*.java

- 2026-04-16 错误：mvn clean install 报错：依赖版本冲突
  原因：spring-boot-starter-parent 版本不一致
  修正：统一使用 bom/import-bom/pom.xml 中的版本，遵循项目规范
```

## 自动初始化

技能首次加载时会自动：

1. 创建 `.trae/learnings/` 目录
2. 创建三个学习记录文件
3. 无需用户手动配置
