## 发票验真状态机定义

```
新增待验真 ──发票验真成功──→ 发票验真通过
新增待验真 ──发票验真失败──→ 发票验真失败
新增待验真 ──强制通过──→ 发票强制通过
修改待验真 ──发票验真成功──→ 发票验真通过
修改待验真 ──发票验真失败──→ 发票验真失败
修改待验真 ──强制通过──→ 发票强制通过
发票验真失败 ──发票验真成功──→ 发票验真通过
发票验真失败 ──强制通过──→ 发票强制通过
发票强制通过 ──发票验真──→ 发票验真通过/发票验真失败

状态守卫：
- 发票验真通过、已红冲、已作废 → 不允许发票验真
- 发票验真通过、已红冲、已作废 → 不允许强制通过
```

| 验真状态 | 允许发票验真 | 允许强制通过 |
|---------|:----------:|:----------:|
| 新增待验真 | Y | Y |
| 修改待验真 | Y | Y |
| 发票验真失败 | Y | Y |
| 发票验真通过 | N | N |
| 发票强制通过 | Y | N |
| 已红冲 | N | N |
| 已作废 | N | N |
