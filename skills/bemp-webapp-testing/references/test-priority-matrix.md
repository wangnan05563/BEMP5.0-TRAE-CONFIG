# BEMP 票据交易管理平台 - 测试范围与优先级矩阵

> 本文档定义了 BEMP 系统各功能模块的测试优先级和风险等级，作为测试用例编写和资源分配的基准。

## 一、优先级定义

| 优先级 | 含义 | 测试策略 |
|--------|------|----------|
| P0 | 核心资金流程，出错导致资金损失或业务中断 | 必须全量测试，覆盖所有边界和异常场景 |
| P1 | 核心业务流程，出错影响业务连续性 | 重点测试主流程和关键异常分支 |
| P2 | 辅助功能，出错影响用户体验 | 常规测试主流程和常见异常 |
| P3 | 管理类功能，出错影响范围有限 | 基础功能验证即可 |

## 二、风险等级定义

| 风险等级 | 含义 |
|----------|------|
| 高 | 可能导致资金损失、业务中断或数据不一致 |
| 中 | 可能导致业务流程异常但可恢复 |
| 低 | 仅影响用户体验或数据展示 |

## 四、高风险场景汇总

| 风险场景 | 涉及模块 | 具体风险 |
|----------|----------|----------|
| 金额精度与利息计算 | 贴现、再贴现、回购、清算 | 浮点数精度丢失、四舍五入规则不一致 |
| 并发操作同一票据 | 质押/解质押、签收/拒绝、买入/卖出 | 竞态条件导致状态不一致 |
| 票据状态机流转 | 承兑→贴现→质押→解付→追索 | 非法状态跳转、状态回退异常 |
| 复核流程中断 | 所有复核类操作 | 复核超时、复核人不存在、自审自批 |
| 报文交互超时 | CPES/ECDS报文收发 | 超时后业务状态不一致、重复报文 |
| 权限越权操作 | 柜员管理、审批管理、交易操作 | 角色变更未生效、跨机构数据泄露 |
| 日终批量处理 | 清算结算、对账、余额结转 | 批量操作原子性、部分失败回滚 |
| 大额交易处理 | 支付、清算、扣款 | 大额支付双人复核、大额清算排队 |
