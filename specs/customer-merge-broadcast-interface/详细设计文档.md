# 客户号合并广播接口详细设计文档

## 1. 功能概述

### 1.1 功能描述
客户号合并广播接口用于接收ECIF系统广播的机构客户合并信息,将XML格式的报文转换为JSON格式,调用产品服务接口完成客户合并操作,并将处理结果返回给ECIF系统。

### 1.2 业务场景
- ECIF系统完成机构客户合并后,通过广播方式通知下游系统
- BEMP系统接收广播报文,提取保留客户和被合并客户信息
- 调用产品服务接口执行客户合并操作
- 返回处理结果给ECIF系统

### 1.3 接口标识
- 外围接口交易码:0402006
- 产品服务接口:PICE070701(Ecif4001Service)

## 2. 接口设计

### 2.1 外围接口规范(0402006广播机构客户合并)

#### 2.1.1 请求报文结构
```
transaction
├── header
│   ├── txCode (交易编号)
│   ├── tellerNo (柜员号)
│   ├── orgCode (机构码)
│   └── isCust (是否客户操作)
└── body
    └── request
        ├── custNo (保留客户号)
        ├── mOrgCust[] (客户基本信息数组)
        │   ├── custNo (客户号)
        │   ├── custSubtype (客户细分类型)
        │   ├── custNmcn (客户中文名)
        │   ├── indusCls (行业类别)
        │   ├── lrCertType (法人代表证件种类)
        │   ├── lrCertNo (法人代表证件号码)
        │   └── lrName (法人代表名称)
        ├── mOrgCertInfo[] (证件信息数组)
        │   ├── custNo (客户号)
        │   ├── certType (证件类型)
        │   ├── certNo (证件号码)
        │   └── primaryCertFlg (主辅证件标志)
        ├── mAddrInfo[] (地址信息数组)
        │   ├── custNo (客户号)
        │   ├── addrType (地址类型)
        │   ├── addrCntt (地址内容)
        │   ├── telNo (电话号码)
        │   ├── provinceCode (省代码)
        │   ├── cityCode (市代码)
        │   ├── prefectureCode (县代码)
        │   └── streetCode (乡镇/街道)
        └── suspectCustNo (被合并客户号)
```

#### 2.1.2 响应报文结构
```
transaction
├── header
│   ├── retCode (返回码)
│   └── retMsg (返回消息)
└── body
    └── response
        ├── custNo (保留客户号)
        ├── mrgdCustNo (被合并客户号)
        ├── chkRsltRetCd (检查结果返回码)
        └── chkRsltRetRsn (检查结果返回原因)
```

## 3. 字段映射关系表

### 3.1 基本字段映射

| 外围字段(XML) | 产品字段(JSON) | 映射说明 |
|---------------|----------------|----------|
| body/request/custNo | requestDto.custNo | 保留客户号,直接映射 |
| body/request/suspectCustNo | requestDto.mrgdCustNo | 被合并客户号,直接映射 |
| - | requestDto.operType | 操作类型,固定值"2"(合并) |

### 3.2 数组字段映射

#### 3.2.1 mOrgCust数组映射(客户基本信息)
| 外围字段(XML) | 产品字段(JSON) | 映射逻辑 |
|---------------|----------------|----------|
| mOrgCust[custNo==保留客户号]/custNmcn | requestDto.custName | 遍历数组,匹配保留客户号,提取客户中文名 |

#### 3.2.2 mOrgCertInfo数组映射(证件信息)
| 外围字段(XML) | 产品字段(JSON) | 映射逻辑 |
|---------------|----------------|----------|
| mOrgCertInfo[custNo==保留客户号 && primaryCertFlg=="1"]/certType | requestDto.certType | 遍历数组,匹配保留客户号且主证件标志为"1",提取证件类型 |
| mOrgCertInfo[custNo==保留客户号 && primaryCertFlg=="1"]/certNo | requestDto.certNo | 遍历数组,匹配保留客户号且主证件标志为"1",提取证件号码 |

#### 3.2.3 mAddrInfo数组映射(地址信息)
| 外围字段(XML) | 产品字段(JSON) | 映射逻辑 |
|---------------|----------------|----------|
| mAddrInfo[] | - | 当前版本仅解析不映射,预留扩展 |

## 4. 处理流程

### 4.1 fromMessage处理流程(XML转JSON)

```
开始
  ↓
接收XML广播报文
  ↓
解析XML根节点
  ↓
提取基本字段
  ├─ custNo(保留客户号)
  └─ suspectCustNo(被合并客户号)
  ↓
解析mOrgCust数组
  ├─ 遍历数组元素
  ├─ 匹配保留客户号
  └─ 提取客户中文名(custNmcn)
  ↓
解析mOrgCertInfo数组
  ├─ 遍历数组元素
  ├─ 匹配保留客户号
  ├─ 匹配主证件标志(primaryCertFlg=="1")
  └─ 提取证件类型和证件号码
  ↓
解析mAddrInfo数组
  └─ 记录日志(预留扩展)
  ↓
构造请求DTO
  ├─ 设置基本字段
  ├─ 设置数组字段提取值
  └─ 设置默认值
  ↓
封装系统报文头
  ↓
返回JSON请求对象
  ↓
结束
```

### 4.2 toMessage处理流程(JSON转XML)

```
开始
  ↓
接收JSON响应数据
  ↓
获取原始请求XML文档
  ↓
创建XML响应构建器
  ↓
构造响应报文头
  ├─ 从原始请求提取header
  ├─ 设置返回码(retCode)
  └─ 设置返回消息(retMsg)
  ↓
构造响应报文体
  ├─ 添加检查结果返回码(chkRsltRetCd)
  ├─ 添加检查结果返回原因(chkRsltRetRsn)
  ├─ 添加保留客户号(custNo)
  └─ 添加被合并客户号(mrgdCustNo)
  ↓
生成XML响应报文
  ↓
返回Message对象
  ↓
结束
```

### 5.2 XML解析关键逻辑

#### 5.2.1 基本字段解析
```java
// 解析基本字段
String custNo = XmlUtil.getNodeValue(requestNode, "custNo");
String suspectCustNo = XmlUtil.getNodeValue(requestNode, "suspectCustNo");
```

#### 5.2.2 数组字段解析
```java
// 解析mOrgCust数组
List<XmlNode> mOrgCustArray = requestNode.getSubNodes("mOrgCust");
if (mOrgCustArray != null && !mOrgCustArray.isEmpty()) {
    for (XmlNode mOrgCust : mOrgCustArray) {
        String mOrgCustNo = XmlUtil.getNodeValue(mOrgCust, "custNo");
        if (custNo != null && custNo.equals(mOrgCustNo)) {
            custName = XmlUtil.getNodeValue(mOrgCust, "custNmcn");
            break;
        }
    }
}
```

#### 5.2.3 主证件提取
```java
// 解析mOrgCertInfo数组,提取主证件
List<XmlNode> mOrgCertInfoArray = requestNode.getSubNodes("mOrgCertInfo");
if (mOrgCertInfoArray != null && !mOrgCertInfoArray.isEmpty()) {
    for (XmlNode mOrgCertInfo : mOrgCertInfoArray) {
        String certCustNo = XmlUtil.getNodeValue(mOrgCertInfo, "custNo");
        String primaryCertFlg = XmlUtil.getNodeValue(mOrgCertInfo, "primaryCertFlg");
        // 匹配保留客户号且为主证件
        if (custNo != null && custNo.equals(certCustNo) && "1".equals(primaryCertFlg)) {
            certType = XmlUtil.getNodeValue(mOrgCertInfo, "certType");
            certNo = XmlUtil.getNodeValue(mOrgCertInfo, "certNo");
            break;
        }
    }
}
```

### 5.3 JSON构造关键逻辑

## 6. 异常处理机制

### 6.1 异常类型

| 异常类型 | 触发场景 | 处理方式 |
|----------|----------|----------|
| XML解析异常 | XML报文格式错误 | 记录错误日志,返回错误响应 |
| 字段缺失异常 | 必填字段为空 | 记录警告日志,使用默认值 |
| 数组遍历异常 | 数组元素为空 | 跳过空元素,继续处理 |
| 类型转换异常 | 字段类型不匹配 | 记录错误日志,返回错误响应 |

### 6.2 异常处理策略

```java
// 字段空值判断
if (custNo != null && custNo.equals(mOrgCustNo)) {
    // 匹配成功
}
```

#### 6.2.2 默认值兜底

```java
// 设置默认值
```

## 7. 日志记录规范

### 7.1 日志级别

| 日志级别 | 使用场景 | 示例 |
|----------|----------|------|
| DEBUG | 调试信息,数组大小等 | 数组元素数量记录 |
| INFO | 关键业务信息 | 请求/响应JSON记录 |
| WARN | 警告信息 | 字段缺失但可继续处理 |
| ERROR | 错误信息 | 解析失败,处理异常 |

### 7.2 日志记录点

#### 7.2.1 请求处理日志
```java
// 记录请求JSON
logger.info("PICE070701MessageConverter请求json: {}", req);
```

#### 7.2.2 响应处理日志
```java
// 记录响应JSON
logger.info("PICE070701MessageConverter响应json", jsonObject);
```

#### 7.2.3 调试日志
```java
// 记录数组解析信息
logger.debug("mAddrInfo数组大小: {}", mAddrInfoArray.size());
```

### 7.3 日志格式规范
- 使用占位符方式记录日志,避免字符串拼接
- 日志内容包含关键业务字段(客户号、操作类型等)
- 异常日志需包含异常堆栈信息

### 7.4 敏感信息处理
- 证件号码等敏感信息在日志中脱敏处理
- 客户名称等敏感信息在日志中脱敏处理
- 禁止在日志中输出完整的XML报文

## 8. 性能优化建议

### 8.1 数组遍历优化
- 使用break提前退出循环,避免不必要的遍历
- 数组为空时快速返回,避免空指针异常

### 8.2 对象创建优化
- 复用JSONObject对象,减少对象创建
- 使用常量字符串,避免重复创建

### 8.3 日志输出优化
- 合理使用日志级别,避免过多DEBUG日志
- 使用占位符方式,避免不必要的字符串拼接
