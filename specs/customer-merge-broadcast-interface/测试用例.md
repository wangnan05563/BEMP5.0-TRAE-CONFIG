# 客户号合并广播接口测试用例文档

## 1. 测试环境说明

### 1.1 测试环境配置
- 操作系统:Windows/Linux
- JDK版本:1.8+
- 应用服务器:Tomcat/WebLogic
- 数据库:Oracle/MySQL
- 中间件:消息队列(ActiveMQ/RabbitMQ)

### 1.2 测试工具
- 接口测试工具:Postman/SOAP UI
- 日志查看工具:日志服务器/本地日志文件
- 数据库工具:PLSQL Developer/Navicat
- XML验证工具:XMLSpy/在线XML验证器

### 1.3 测试数据准备
- 测试客户号:有效的机构客户号(13位)
- 测试证件信息:有效的证件类型和证件号码
- 测试地址信息:完整的地址信息(省市区街道)

## 2. 测试场景列表

### 2.1 正常场景测试用例

#### TC001:正常客户合并广播处理
**测试目的**:验证正常客户合并广播报文的处理流程

**前置条件**:
- ECIF系统已发送客户合并广播报文
- BEMP系统消息队列正常接收消息
- 产品服务接口正常可用

**测试步骤**:
1. ECIF系统发送客户合并广播报文
2. BEMP系统接收XML报文
3. 解析XML报文提取客户信息
4. 构造JSON请求调用产品服务
5. 产品服务处理客户合并
6. 构造XML响应返回ECIF系统

**测试数据**:
```xml
<transaction>
  <header>
    <txCode>0402006</txCode>
    <tellerNo>TELLER001</tellerNo>
    <orgCode>ORG001</orgCode>
    <isCust>1</isCust>
  </header>
  <body>
    <request>
      <custNo>C123456789012</custNo>
      <mOrgCust>
        <custNo>C123456789012</custNo>
        <custSubtype>001</custSubtype>
        <custNmcn>测试企业客户A</custNmcn>
        <indusCls>0001</indusCls>
        <lrCertType>01</lrCertType>
        <lrCertNo>110101199001011234</lrCertNo>
        <lrName>张三</lrName>
      </mOrgCust>
      <mOrgCust>
        <custNo>C987654321098</custNo>
        <custSubtype>001</custSubtype>
        <custNmcn>测试企业客户B</custNmcn>
        <indusCls>0002</indusCls>
        <lrCertType>01</lrCertType>
        <lrCertNo>110101199002021234</lrCertNo>
        <lrName>李四</lrName>
      </mOrgCust>
      <mOrgCertInfo>
        <custNo>C123456789012</custNo>
        <certType>01</certType>
        <certNo>91110000123456789X</certNo>
        <primaryCertFlg>1</primaryCertFlg>
      </mOrgCertInfo>
      <mOrgCertInfo>
        <custNo>C123456789012</custNo>
        <certType>02</certType>
        <certNo>123456789</certNo>
        <primaryCertFlg>0</primaryCertFlg>
      </mOrgCertInfo>
      <mOrgCertInfo>
        <custNo>C987654321098</custNo>
        <certType>01</certType>
        <certNo>91110000987654321Y</certNo>
        <primaryCertFlg>1</primaryCertFlg>
      </mOrgCertInfo>
      <mAddrInfo>
        <custNo>C123456789012</custNo>
        <addrType>01</addrType>
        <addrCntt>北京市海淀区中关村大街1号</addrCntt>
        <telNo>010-12345678</telNo>
        <provinceCode>110000</provinceCode>
        <cityCode>110100</cityCode>
        <prefectureCode>110108</prefectureCode>
        <streetCode>110108001</streetCode>
      </mAddrInfo>
      <suspectCustNo>C987654321098</suspectCustNo>
    </request>
  </body>
</transaction>
```

**预期结果**:
1. XML报文解析成功
2. 提取保留客户号:`C123456789012`
3. 提取被合并客户号:`C987654321098`
4. 提取保留客户名称:`测试企业客户A`
5. 提取主证件类型:`01`
6. 提取主证件号码:`91110000123456789X`
7. 构造JSON请求成功
8. 产品服务调用成功
9. 返回XML响应报文包含正确的客户号和检查结果

**验证要点**:
- [ ] XML解析无异常
- [ ] 客户号提取正确
- [ ] 客户名称提取正确
- [ ] 主证件信息提取正确
- [ ] JSON构造正确
- [ ] 产品服务调用成功
- [ ] XML响应构造正确
- [ ] 日志记录完整

## 10. 测试注意事项

### 10.1 数据安全
- 测试数据不得包含真实客户信息
- 敏感信息需脱敏处理
- 测试完成后清理测试数据

### 10.2 环境隔离
- 测试环境与生产环境隔离
- 测试数据库独立
- 测试消息队列独立

### 10.3 日志管理
- 测试期间开启DEBUG日志
- 测试完成后恢复INFO日志
- 日志文件定期清理

### 10.4 问题跟踪
- 发现问题及时记录
- 问题修复后需回归测试
- 严重问题需升级处理
